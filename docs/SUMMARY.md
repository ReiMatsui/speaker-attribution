# 現状サマリー（2026-06-09）

## ゴール（不変）
対面会議を、低遅延・リアルタイムで「誰が何を言ったか」に文字起こしする。
主軸は1人1マイク。1本マイクは手軽な代替案として保持。

## 一言でいう現状
**「最高精度の既製ストリーミングASR（parakeet-mlx）の前段に、本研究の"漏れ込み除去ゲート"を
差し込む」アーキテクチャが、実機（M4 Pro）で有効と実証できた。** ASRは既製品を借り、
自前で作るのは漏れ込み除去ゲートのみ。リアルタイムの道筋も通った。

## できたこと

**1. 動くパイプライン（uv管理・テスト通過）**
VAD → チャンネル間相関(GCC-PHAT)で漏れ込み判定 → （任意で声紋照合）→ 本人時間以外を
無音化した連続音声 → ASR。

**2. 信頼できる評価へ移行**
自前合成正解・Whisper出力正解（動作確認止まり）から、**LibriSpeech公式トランスクリプト
（人手参照）** でのWER評価に移行。

**3. 主要な実験結果（漏れ込み-12dB、公式参照、Whisper base）**

| 取り組み | WER |
|---|---|
| 初期（断片ASR） | 30% |
| 連続マスク方式に変更 | 23% |
| ゲートをrecall優先に緩和 | 17% |
| マージ/パッド拡大 | 13% |
| （クリーン下限） | 4% |

効いたのは「音を切らず連続でASRし、時間で取捨選択する」こと。逆に「音を洗浄する」発想
（時間周波数マスク・最小二乗キャンセル）は軒並み外れ（=失敗として記録）。

**4. 漏れ込み量で価値が変わると特性化**

| 漏れ込み | ゲートなし | ゲートあり |
|---|---|---|
| -12dB(弱) | 6% | 13% |
| -6dB(中) | 130%(崩壊) | 14% |
| 0dB(等音量) | 278% | 233% |

中程度の漏れ込みでゲートが効く。0dB等音量は対面では起きない最難ストレステスト。

**5. リアルタイム化の壁を実測**
素朴なチャンク化は崩壊（1秒チャンクでWER 44%、オフライン13%）。ただし**計算は律速でない**
（RTF<1）。壁は「文脈」。突破口はローリング文脈＋ストリーミングASR。

**6. 1本マイク vs 1人1マイク**
ターンテイキング会話で 1本21% / 1人1マイク13%。1本でも実用圏だが同時発話に弱い。

**7. 既製品(parakeet-mlx)統合を実機実証（核心）**
- クリーンな各ch → ほぼ完璧にリアルタイムwho-said-what（計算遅延384ms）。
- **漏れ込みが入ると-20dBの静かな漏れでも崩壊**（優秀なASRほどかすかな漏れまで文字化）。
- **本研究のゲートを通すと各chが本人だけに復活**。崩壊→復活を実機で確認。

**8. ゲートの誤判定バグを修正（最新）**
「漏れだけの時間帯」を誤って本人扱いする問題を、全チャンネル対の「自分は他者の遅れたコピーか」
判定に直して解消。チャンネル間の混入がほぼ消えた。

## 確定したアーキテクチャ
各チャンネルごとに **［本研究のゲート（漏れ込み除去）］→［parakeet-mlx（既製ストリーミングASR）］**。

## 残課題
1. ゲートのストリーミング化 → parakeetとリアルタイム統合。
2. 境界のわずかな残留漏れ・recall取りこぼしの低減。
3. **AMIコーパスでの現実検証**（合成漏れ込みは"きれいすぎ"の可能性。実環境の漏れ込み量と性能を測る）。
4. cpWERでの相槌・割り込みの当て分け評価。

## 正直な注意（妥当性）
合成データ（反響なし・線形漏れ込み）、英語、小規模サンプル、実漏れ込み量は未grounding。
これらを実データで固めるのが信頼性の次の一歩。
